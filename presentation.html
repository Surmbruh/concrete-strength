Прогнозирование прочности бетона
с использованием архитектуры
GAN + NEAT + BNN

Королева Мария · Долгих Данил

Представляем подход к прогнозированию прочности бетонной смеси на основе Generative Adversarial Network. Генератор — полносвязная сеть с residual connections — предсказывает прочность и оценку неопределённости. Дискриминатор — сеть с топологией, эволюционированной через NEAT, и байесовскими весами через Pyro SVI.

Аннотация

Генератор — полносвязная сеть с residual connections — предсказывает прочность $\mu$ и оценку неопределённости $\sigma$ по составу бетонной смеси.

Дискриминатор — нейросеть с топологией, эволюционированной через NEAT, и байесовскими весами (BNN через Pyro SVI) — оценивает правдоподобность предсказаний генератора.

Основная метрика: MAE при $t = 28$ дней

3. Архитектура: ConcreteGAN

Состав (10 признаков) │ ▼ ┌──────────────┐ (μ, σ) │ ГЕНЕРАТОР │────────────────┐ │ (ResNet FC) │ │ └──────────────┘ ▼ ┌─────────────────┐ Реальные данные ────▶│ ДИСКРИМИНАТОР │─▶ real/fake (состав, прочность) │ (NEAT + BNN) │ └─────────────────┘ │ adversarial loss │ обновление G
Conditional GAN для регрессии. Генератор принимает не шум, а вектор состава бетона и предсказывает пару (μ, σ). Дискриминатор получает пару (состав, прочность) и оценивает правдоподобность. Adversarial loss действует как дополнительный регуляризатор к supervised loss.

3.2 Генератор (ConcreteGenerator)

Input(10) → FC(256) + ResBlock(256) → FC(128) + ResBlock(128) → FC(64) + ResBlock(64) → Linear(64 → 2) → (μ, σ) 219 714 параметров BatchNorm · LeakyReLU · Dropout(0.1)

Residual connections

$\text{output} = F(x) + x$ — каждый слой учит инкрементальное уточнение

Loss: Gaussian NLL

$$\mathcal{L} = \frac{1}{2}\left[\log(\sigma^2) + \frac{(y - \mu)^2}{\sigma^2}\right]$$

Штрафует и за ошибку $\mu$, и за некалиброванную $\sigma$

Два выхода

$\mu$ — прочность, $\sigma$ — aleatoric uncertainty

Модель предсказывает не только значение прочности μ, но и aleatoric uncertainty σ. Это позволяет модели сказать: «я предсказываю 35 ± 8 МПа для этого нестандартного состава». NLL vs MSE: MSE оптимизирует только μ, NLL находит баланс — если σ слишком мала при большой ошибке, штраф огромен; если σ слишком велика, штраф log(σ²) растёт.

3.3 Дискриминатор (NEAT + BNN)

NEAT — эволюция топологии

  • Популяция 80 геномов, ~8 поколений
  • Мутации: связи, узлы, возмущение весов
  • Видообразование защищает инновационные топологии
  • Результат — оптимизированная архитектура дискриминатора

BNN — байесовские веса (Pyro SVI)

  • Каждый вес w заменяется на q(w) ≈ N(μ_w, σ_w)
  • Prior N(0,1) — implicit L2-регуляризация
  • 3 SVI-шага на каждый шаг дискриминатора
  • Epistemic uncertainty дискриминатора

Обычные GAN нестабильны: дискриминатор быстро становится слишком хорошим. BNN-дискриминатор «мягче» — его неуверенность в оценках стабилизирует adversarial training.

NEAT эволюционирует и топологию (какие нейроны с какими связаны), и веса одновременно. Это даёт топологию, оптимизированную под задачу, вместо ручного подбора. BNN предотвращает ситуацию, когда дискриминатор с 100% уверенностью отличает real от fake — генератор тогда не получает полезных градиентов. Label smoothing (real=0.9) дополнительно стабилизирует.

MC-Dropout (Gal, Ghahramani, 2016): dropout остаётся включённым при инференсе, N=20-50 forward passes. Разброс предсказаний аппроксимирует байесовский posterior. Корреляция σ-error 0.87 означает: когда модель предсказывает высокую σ, она действительно ошибается больше. PICP 97% при 95% CI — модель слегка консервативна, что предпочтительно для инженерных задач.

3.5 Обучение: трёхфазная стратегия

ФазаЭпохиLoss генератора
1. Warmup0–100MSE + NLL
2. Transition100–200MSE + αt × Adversarial (α: 0→0.7)
3. Full GAN200–5000.3 × MSE + 0.7 × Adversarial

Physics-informed regularization

  • Монотонность: прочность растёт с возрастом (∂f/∂t ≥ 0)
  • Закон Абрамса: прочность падает с ростом w/c ratio
  • ГОСТ ограничения: предсказания в разумных пределах
Если генератор ещё ничего не знает, дискриминатор тривиально отличает истинные составы от ложных и генератор получает бесполезные градиенты. Warmup даёт генератору базовое качество, с которого adversarial training может улучшить его дальше. Physics loss — дополнительный регуляризатор, штрафующий за нефизичные предсказания.

4.3 Ансамбль

Разные модели ошибаются в разных местах. Stacking: предсказания как входы для мета-модели. OOF MAE ≈ Test MAE.

Supervised сначала: GAN добавляет гиперпараметры (lr D, balance G/D, physics weights). Если генератор плохо настроен — GAN не поможет. Стекинг: GBR (n_estimators=50, max_depth=2). K-Fold CV: каждый сэмпл предсказан моделью, которая его не видела. OOF MAE=4.44, Test MAE=5.03, gap 0.59 — приемлемый.
